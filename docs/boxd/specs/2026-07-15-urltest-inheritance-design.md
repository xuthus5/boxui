# URLTest 全局默认与订阅覆盖设计

## 目标

为订阅自动生成的 sing-box `urltest` 出站组提供一套全局默认参数，并允许创建或编辑订阅时按字段覆盖。订阅未配置的字段始终继承当前全局默认值，避免复制默认值后失去联动。

本功能只管理 sing-box `urltest` 出站组，不改变节点卡片上的 TCP、HTTP、ICMP 测速地址与测速流程。

## 方案选择

### 方案 A：创建订阅时复制全局参数

实现简单，但全局默认值修改后既有订阅不会跟随变化，用户也无法区分字段是继承值还是历史副本。

### 方案 B：保存可选覆盖并在同步时动态解析（采用）

全局设置保存完整默认值；订阅只保存显式覆盖字段。每次同步配置时先读取当前全局设置，再用订阅覆盖合并，生成最终 `urltest` 出站组。该方案能保持继承关系，兼容旧数据，且不会增加另一套分组模型。

### 方案 C：直接保存完整 sing-box 分组 JSON

灵活度最高，但会把普通订阅设置暴露成高级配置编辑，校验、迁移和前端交互都更复杂，不符合当前需求的最小充分范围。

## 配置模型

### 全局默认值

应用设置新增“订阅 URLTest 默认值”配置：

- `enabled`：是否默认为订阅创建 URLTest 组，默认 `true`。
- `url`：测速 URL，默认 `https://www.gstatic.com/generate_204`。
- `interval`：测试间隔，使用 sing-box 时长格式，默认 `3m`。
- `tolerance`：切换容差，单位毫秒，默认 `50`，范围 `0..65535`。

这组配置与现有“系统测速地址”保持独立。后者继续只服务面板 TCP、HTTP、ICMP 测速。

### 订阅覆盖值

订阅保存一个可选 `urltest` 覆盖对象，其中每个字段均可缺省：

```json
{
  "urltest": {
    "enabled": false,
    "url": "https://example.com/generate_204",
    "interval": "5m",
    "tolerance": 100
  }
}
```

- 整个对象缺省：所有字段继承全局默认值。
- 对象存在但某字段缺省：该字段继承全局默认值。
- `enabled: false`：该订阅不生成 URLTest 组。
- `enabled: true`：即使全局默认关闭，该订阅仍生成 URLTest 组。
- `tolerance: 0` 是有效显式覆盖，不能被当作缺省值。

旧订阅没有 `urltest` 字段，反序列化后自然继承全局默认值，无需数据库迁移。

## 后端设计

### 设置 API

新增：

- `GET /api/settings/urltest-defaults`
- `PUT /api/settings/urltest-defaults`

响应和请求使用完整的全局配置对象。设置以 JSON 写入现有 bbolt `settings` bucket，读取不到数据时返回内置默认值。

服务端校验：

- URL 必须是绝对的 HTTP 或 HTTPS URL。
- `interval` 必须是大于零的有效时长。
- `tolerance` 必须在 `0..65535` 内。
- 请求体无效时返回现有统一的 `invalid_request` 错误结构。

### 订阅 API

扩展现有订阅创建、读取、列表与更新结构，增加可选 `urltest` 覆盖对象。创建和更新均校验显式提供的字段；未提供的字段不写入覆盖。

更新请求以提交的完整覆盖对象替换旧覆盖对象：

- 缺省或 `null` 表示清除全部覆盖，恢复全局继承。
- 对象中的缺省字段表示该字段恢复全局继承。

### 配置同步

同步每个订阅时执行：

1. 读取当前全局 URLTest 默认值。
2. 将订阅的非空覆盖字段合并到默认值。
3. 若最终 `enabled` 为 `false`，不生成该订阅的 `urltest` 组，也不把该组加入 `proxy` selector。
4. 若最终 `enabled` 为 `true`，使用最终 URL、间隔和容差生成分组。
5. 回收此前由该订阅生成、但现在已关闭的旧 URLTest 组。

保存全局设置或订阅设置后，前端调用现有 `/api/nodes/sync-config`，使生成配置立即更新；同步失败时显示错误，不提示保存与应用全部成功。

## 前端设计

### 应用设置

在现有运行设置区域新增独立的“订阅 URLTest 默认值”卡片，使用已安装的 shadcn `Card`、`Field`、`Input`、`Switch` 和 `Button`：

- 默认启用开关。
- 测试 URL 输入框。
- 测试间隔输入框，并提示示例 `3m`、`30s`。
- 容差数字输入框，标明单位为毫秒。
- 说明文字明确区分“订阅 URLTest”和“节点 TCP/HTTP/ICMP 测速”。

不新增全局 CSS，不修改 shadcn 基础组件样式。

### 订阅表单

创建和编辑订阅的 Dialog 新增 URLTest 区域：

- URLTest 组策略使用“继承全局 / 启用 / 关闭”三态选择，默认继承全局。
- URL、间隔和容差输入框均允许留空；留空表示继承全局，并用占位提示展示当前全局值。
- 用户只需填写需要覆盖的字段，其他字段继续跟随全局默认值。
- 提供“恢复全部默认值”操作，提交 `urltest: null` 清除旧覆盖。

订阅卡片用简短 Badge 表示“URLTest：继承 / 自定义 / 已关闭”，不展示完整参数，避免卡片信息过载。

### 节点页面

节点页面不增加 URLTest 参数配置。节点测速继续使用现有卡片测速能力；现有运行时组只承担当前状态、Selector 切换和立即执行 URLTest，不承担参数编辑。

## 数据流

```text
应用设置 ──保存全局默认值──┐
                           ├──> 同步配置 ──> 解析继承/覆盖 ──> sing-box outbounds
订阅表单 ──保存可选覆盖────┘
```

全局默认值发生变化后，所有仍继承对应字段的订阅在下一次同步时使用新值；有显式覆盖的字段保持不变。

## 错误处理

- 后端拒绝非法 URL、非正时长和超范围容差，不把无效配置写入数据库。
- bbolt 读取、编码和写入错误均向上传递并返回统一 API 错误。
- 前端保存设置后若配置同步失败，显示同步错误；表单保留用户输入以便修正或重试。
- 旧数据、设置键缺失或订阅覆盖缺失均回退到内置默认值。

## 测试设计

### Go

- 全局默认值读取、保存、设置缺失时回退默认值、非法持久化数据时报错。
- 设置 API 的正常路径及 URL、时长、容差错误路径。
- 订阅创建、更新、清除覆盖、`false` 和 `0` 等零值覆盖。
- 继承全部字段、部分字段覆盖、全局关闭后订阅显式开启、订阅显式关闭。
- 关闭订阅 URLTest 后旧分组被回收且 `proxy` selector 不再引用。
- 旧订阅 JSON 无新增字段时保持兼容。

### React/TypeScript

- 应用设置正确加载、编辑、保存全局默认值并触发配置同步。
- 订阅表单默认继承，按字段留空继承并只提交显式覆盖。
- 编辑自定义订阅时恢复覆盖值，重新继承时提交 `null`。
- 非法输入阻止提交并显示明确提示。
- 中英文文案及移动端 Dialog 内容可访问、可滚动。

最终执行 Go 与前端覆盖率门禁、lint、生产构建、嵌入资源验证和 systemd 本地部署验证。

## 非目标

- 不合并现有节点 TCP、HTTP、ICMP 测速地址与 sing-box URLTest URL。
- 不开放 `idle_timeout`、`interrupt_exist_connections` 等额外高级参数。
- 不允许在节点页面编辑 URLTest 参数。
- 不修改 sing-box 完整配置编辑器的现有行为。
